# 📝 Code Review - Rename Ebook Tasks and Add html2md

이 리뷰 문서는 `apps/ebook` 디렉토리 내의 Poe 태스크 명칭 변경 및 신규 `html2md` 변환기(이미지, CSS 분리 및 문장 병합 포함) 구현에 대한 코드 품질 검증 사항을 다룹니다.

## 1. 아키텍처 및 설계 검증
- **명칭 일관성**: 사용자가 요청한 타겟 명칭(`pdf2md`, `pdf2html`, `translate_md`, `html2md`)이 `Makefile`, `pyproject.toml`, `process.py`에 일관되게 반영되었습니다.
- **모듈 분리**: HTML에서 마크다운으로의 변환 처리를 담당하는 `HTMLToMarkdownConverter` 클래스를 `html_to_markdown.py`로 별도 분리하여 OOP 및 단일 책임 원칙(SRP)을 준수하였습니다.

## 2. 보안 및 리소스 관리 검증
- **이미지 분리**: base64 인라인 데이터 형식의 대형 이미지 파일들을 PNG 파일로 추출하여 `images/` 하위 경로에 저장하고 마크다운에서는 상대 경로(`![image](images/...)`)로 링크함으로써 깃 형상 관리의 팽창을 방지하고 가독성을 높였습니다.
- **스타일 분리/제거**: `<style>` 및 `<script>` 요소를 마크다운 변환 전에 DOM 트리 수준에서 `decompose` 처리하여 CSS 텍스트 코드가 마크다운에 그대로 노출되는 버그를 원천 차단했습니다.

## 3. 안정성 및 예외 처리 검증
- **문장 복원 알고리즘**: PDF to HTML 변환으로 인해 잘게 쪼개진 단락 간 싱글 개행 문자(`\n`)를 앞뒤 구두점 및 대소문자 패턴을 분석하여 하나의 완성된 문장으로 매끄럽게 병합하고, 마크다운의 구조적 기호(헤더, 코드 블록, 리스트 등)는 훼손하지 않도록 안전 장치를 구성했습니다.
- **파일명 위생 처리(Sanitization)**: 이미지 파일 생성 시 파일명에 들어있는 공백 문자를 언더스코어(`_`)로 치환하여 저장함으로써 마크다운 링크 문법 깨짐 및 인코딩 버그를 차단했습니다.
