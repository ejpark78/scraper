# 📋 계획서: 아카이빙 시 INDEX.md 자동 갱신 자동화 계획 (055-automate-index-md-on-archive.plan.md)

## 1. 🔍 문제 정의 및 분석
- **현상**: `make agents-squash` 실행 후 `.archive.md`로 개별 파일들이 병합 및 제거될 때, `INDEX.md`에는 삭제된 파일들이 여전히 링크로 남아 깨지는 버그가 존재함.
- **목적**: 아카이빙 작업이 완료된 후 스크립트(`squash-artifacts.sh`)가 `docs/artifacts/` 내의 파일들을 동적으로 스캔하여, 물리적으로 존재하는 아카이브 및 개별 아티팩트에 맞춰 **`INDEX.md`를 자동으로 정밀하게 갱신/재생성**하도록 코드에 반영.

---

## 🛠️ 해결 방안
1. **`squash-artifacts.sh` 내 `INDEX.md` 동적 갱신 로직 탑재**:
   - `run_archive` 함수 성공 후 자동으로 실행되는 `update_index_md` 함수를 스크립트에 추가합니다.
   - `update_index_md`는 `docs/artifacts/`에 남아 있는 실물 아티팩트들을 분석하여 `INDEX.md`를 Overwrite 생성합니다:
     - 물리적으로 존재하는 `*.archive.md` 파일들은 각각 하나의 요약 행으로 구성.
     - 아직 10개 단위가 차지 않아 아카이빙되지 않은 개별 아티팩트(`.plan.md`, `.adr.md`, `.spec.md` 등)들은 개별 행으로 순차 정렬.
2. **Bash 스크립트 기반 `INDEX.md` 생성 로직**:
   - 아카이브 표와 개별 파일 표를 통합하여 실시간 현상태를 마크다운 표로 깔끔하게 렌더링.

### 1. `update_index_md` 자동화 함수 탑재
- **수정 내용**:
  - `squash-artifacts.sh` 스크립트 실행 제어 흐름 말미에 `update_index_md` 함수를 연계 호출하도록 구성.
  - `find` 명령어를 통해 실물 `*.archive.md` 아카이브와 `[0-9][0-9][0-9]-*.md` 개별 아티팩트를 실시간 동적 검색.
  - `grep -c`를 활용하여 각 아카이브 내에 몇 개의 개별 문서 헤더(`## [0-9][0-9][0-9]-`)가 포함되어 있는지 동적으로 파악하여 파일 개수 정보의 정확도 보장.
  - 마크다운 표 구조를 생성하여 `docs/artifacts/INDEX.md`에 실시간으로 오버라이트 작성.
- **결과**: `make agents-squash` 실행 후 발생하는 깨진 파일 링크 오류가 완전히 제거되고, 물리적 아티팩트 보존 상태와 `INDEX.md`가 100% 동기화됨.
