# 📋 작업 계획서: 아카이브 파일 덮어쓰기 버그 해결 및 유실 데이터 복구 (077-fix-archive-overwrite-issue.plan.md)

이 문서는 `make agents-squash` 실행 시 기존에 작성되어 있던 아카이브 파일(`*.archive.md`)을 단순히 `>` 연산자로 덮어써서 기존에 저장된 다른 아티팩트 이력(071~075)이 지워지던 설계상 버그를 긴급 수정하고, Git 이력으로부터 유실된 데이터를 복구하여 병합하는 계획을 수립합니다.

---

## 1. 🔍 문제 상황 및 원인 분석
- **현상**: 
  - 첫 번째 `make agents-squash` 실행으로 `071-075` 이력이 `071-080.archive.md`에 정상 보존됨.
  - 두 번째 `make agents-squash` 실행으로 `076`이 추가될 때, 기존 아카이브 파일(`071-080.archive.md`)이 새 아티팩트(`076`)로만 완전히 덮어써져 `071-075` 이력이 파일에서 사라짐.
- **원인**:
  - `squash-artifacts.sh`의 `run_archive()` 함수 내에서 아카이브 파일 작성 시 추가(Append, `>>`)가 아닌 덮어쓰기(Write, `>`) 연산자를 사용하고 있어 발생한 버그입니다.
  - 기존 아카이브 파일의 내용을 먼저 읽어와 존재하지 않는 새로운 헤더의 이력만 뒤에 덧붙이거나, 안전하게 병합하는 처리가 누락되어 있었습니다.

---

## 2. 🛠️ 해결 방안

### 개선 1: 기존 아카이브 파일이 존재할 경우 증분 병합 (Incremental Merge)
- `run_archive()` 함수에서 `$archive_file`이 이미 존재하는 경우:
  1. 기존 파일 내에 이미 보존되어 있는 피처 헤더들(예: `## 071-joplin-sync-optimization.summary`)을 검사합니다.
  2. 이미 백업되어 있는 피처에 대해서는 중복 병합을 우회합니다.
  3. 존재하지 않는 새로운 피처 파일의 이력만 파일 끝에 추가(`>>`) 방식으로 기록하도록 안전한 병합 로직을 구현합니다.

### 개선 2: Git 이력을 통한 `071-075` 유실 데이터 복구
- 덮어쓰기로 인해 임시로 유실된 `071-080.archive.md` 내의 `071-075` 세부 내역을 직전 Git 커밋에서 안전하게 추출하여 복원합니다.
- 복원한 내용과 현재의 `076` 내역을 병합하여 아카이브의 무결성을 다시 100% 확보합니다.

2. `scripts/agents/squash-artifacts.sh` 내 `run_archive` 함수에서 기존 아카이브 파일이 있는 경우 증분 추가(Append/Merge)로 처리하도록 스크립트 수정.
3. 수정 완료 후 `make agents-squash` 기동 테스트를 통해 중복되지 않고 안전하게 새 아티팩트(`077`)가 덧붙여지는지 검증.
4. 결과보고서(`.walkthrough.md`) 작성 및 최종 커밋.
