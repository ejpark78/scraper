# Gitea & Vikunja 로컬 인프라 구축 결과보고서 (Make 기동 시 초기 계정 생성 자동 연동)

## 변경 사항 및 구성 완료 요약

Gitea 환경 구축을 완료하고, 구동 타겟을 고도화하여 수동 셋업 단계를 통일했습니다.

1. **`make up-gitea` 실행 로직 강화**:
   - 컨테이너가 완전히 실행되어 기동되는 시간을 보장하기 위해 구동 명령 후 `5초`간 대기를 수행합니다.
   - 내부 CLI 진단 도구를 백그라운드로 작동시켜 `admin` 사용자가 존재하는지 실시간 확인합니다.
   - 데이터베이스가 비어있어 사용자가 없을 경우, 아래의 마스터 계정을 자동으로 즉시 생성해 줍니다.
     - **아이디**: `admin`
     - **비밀번호**: `admin12345`
     - **이메일**: `admin@example.com`

---

## 🚀 로컬 명령어 테스트 안내

규칙 1번(임의 셸 실행 금지) 및 12번(환경 제어 공동 위임)에 따라, 명령어 실행은 사용자가 수동으로 실행해주셔야 합니다. 터미널에서 다음 명령어를 실행하면, 계정이 없는 경우 생성까지 완료하여 한 번에 가동됩니다.

```bash
make up-gitea
```
*(성공적으로 기동되면 바로 브라우저를 열고 `admin` / `admin12345` 로 로그인 테스트를 진행해 주시면 됩니다.)*
