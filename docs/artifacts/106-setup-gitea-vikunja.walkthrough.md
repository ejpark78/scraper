# Gitea & Vikunja 로컬 인프라 구축 결과보고서 (계정명 예약어 우회 반영)

## 변경 사항 및 구성 완료 요약

Gitea 내부 보안 제약 조건(Reserved word `admin` 가입 제한)을 극복하고, 메이크 단축 스크립트를 통해 초기 계정을 문제없이 자동 구성할 수 있도록 보완 조치를 완료했습니다.

1. **Gitea 관리자 계정명 교체**:
   - 기존의 `admin` 계정명이 예약어로 제한됨에 따라 관리자 아이디를 **`gitea-admin`**으로 전면 교체하였습니다.
   - `make up-gitea` 수행 시 자동으로 계정 존재 여부를 확인하고 `gitea-admin` / `admin12345` 계정을 데이터베이스에 기입합니다.
     - **아이디**: `gitea-admin`
     - **비밀번호**: `admin12345`

---

## 🚀 로컬 명령어 기동 및 검증 안내

규칙 1번(임의 셸 실행 금지) 및 12번(환경 제어 공동 위임)에 따라, 명령어 실행은 사용자가 수동으로 실행해주셔야 합니다. 터미널에 다시 아래 명령어를 기동해 주세요.

```bash
make up-gitea
```

기동이 성공적으로 끝나면 [Gitea 로그인](https://gitea.localhost/) 페이지에서 `gitea-admin` / `admin12345` 계정으로 바로 진입하실 수 있습니다.
