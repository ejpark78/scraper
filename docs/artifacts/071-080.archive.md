## 🛠️ 변경 전후 코드 대비

### 1. `scripts/agents/squash-artifacts.sh` - `update_index_md()` 개편
아카이브 디렉토리를 갱신할 때 아카이브 표만 그리고 끝내던 것에서, 각 `.archive.md` 내부의 개별 피처 헤더를 grep 파싱하여 앵커 링크 리스트로 시각화 출력하도록 개편했습니다.

#### 변경 전 (Before)
```bash
        for ind in "${individual_files[@]}"; do
            local bname=$(basename "$ind")
            local prefix=$(echo "$bname" | sed -E 's/^([0-9]+)-.*/\1/')
            local desc=$(echo "$bname" | sed -E 's/^[0-9]+-(.*)\.md$/\1/')
            echo "|   | $desc | [$bname]($bname) | 1 |"
        done
    } > "$index_file"
```

#### 변경 후 (After)
```bash
        for ind in "${individual_files[@]}"; do
            local bname=$(basename "$ind")
            local prefix=$(echo "$bname" | sed -E 's/^([0-9]+)-.*/\1/')
            local desc=$(echo "$bname" | sed -E 's/^[0-9]+-(.*)\.md$/\1/')
            echo "|   | $desc | [$bname]($bname) | 1 |"
        done

        echo ""
        echo "## 📂 상세 아카이브 내역 (Archived Details)"
        echo ""

        for arch in "${archives[@]}"; do
            local bname=$(basename "$arch")
            local range=${bname%.archive.md}
            echo "### 📦 아카이브: $range ($bname)"
            echo ""
            
            while IFS= read -r header; do
                if [ -n "$header" ]; then
                    local clean_h=$(echo "$header" | sed -E 's/^## //')
                    # Make anchor slug compatible with markdown spec
                    local slug=$(echo "$clean_h" | tr '[:upper:]' '[:lower:]' | sed -E 's/[^a-z0-9 _-]//g' | tr ' ' '-')
                    echo "- [$clean_h]($bname#$slug)"
                fi
            done < <(grep '^## [0-9][0-9][0-9]-' "$arch" || true)
            echo ""
        done
    } > "$index_file"
```

## 077-fix-archive-overwrite-issue.plan

# 📋 작업 계획서: 아카이브 덮어쓰기 문제 해결 및 Git 탐색 덤프 루프 방지 규칙 추가 (077-fix-archive-overwrite-issue.plan.md)

이 문서는 `make agents-squash` 아티팩트 아카이빙 시 기존 아카이브 파일이 덮어써져 데이터가 유실되는 문제를 해결하기 위해 스크립트를 개선하고, Git 히스토리 탐색 덤프 루프 방지 규칙을 Git Flow 규칙 및 프로젝트 AGENTS.md에 보완하는 계획을 기록합니다.

---

## 1. 🔍 문제 상황 및 필요성
- `squash-artifacts.sh`가 아티팩트를 아카이빙할 때 단순 덮어쓰기(`>`) 방식으로 작성되어 있어, 이전 턴에 squash가 진행되어 이미 원본 파일들이 지워진 상태에서 추가 squash를 실행하면 기존 아카이브 내역이 유실되는 현상이 발생했습니다.
- 에이전트가 Git의 파일 삭제 및 복구 이력을 찾는 과정에서 한 번에 좁은 범위를 파악하지 못하고 여러 턴에 걸쳐 `git log`, `git show`, `git reflog`를 단발적으로 반복 조회하는 비효율성(덤프 루프)이 확인되어 제약 규칙화가 필요합니다.

---

## 2. 🛠️ 해결 방안

### 개선 1: `squash-artifacts.sh` 아카이브 증분 추가 방식 도입
- `run_archive()` 함수 내에서 대상 `$archive_file`이 이미 물리적으로 존재하는지 체크합니다.
- 파일이 없을 경우 헤더를 초기화 생성(`>`)합니다.
- 아카이빙 대상 소스 파일들이 이미 아카이브에 기입되어 있는지 `grep -q "^## $stem"`으로 검사하여 중복을 우회합니다.
- 존재하지 않는 아티팩트 본문만 증분 병합(`>>`)한 뒤 원본을 삭제합니다.

### 개선 2: Git Flow 규칙 및 AGENTS.md에 Git 탐색 덤프 루프 방지 명시
- `.agents/rules/git_flow.md` 파일에 "Git 히스토리 탐색 덤프 루프 방지" 조항을 추가합니다.
- 프로젝트 루트 `AGENTS.md`의 규칙 16번(Git Flow 브랜치 전략 및 에이전트 행동 지침)에 덤프 루프를 돌지 말고 단일 턴에 통합 조회해야 한다는 문장을 보완합니다.
